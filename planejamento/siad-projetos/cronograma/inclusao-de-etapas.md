# Inclusão de etapas

Para cadastrar a etapa de um projeto, acesse o cronograma e clique no botão _<mark style="color:purple;">Cadastrar Etapa</mark>_.&#x20;

<figure><img src="../../../.gitbook/assets/xx4 (2).png" alt=""><figcaption></figcaption></figure>

Após clicar no botão, aparecerá um pop-up na tela com algumas informações necessárias sobre a etapa. São elas:

* Nome da etapa
* Status
* Data de Início
* Data Final
* Área Responsável
* Responsável Principal
* Descrição da Etapa
* Anexos

{% hint style="warning" %}
Todos os campos de preenchimento são obrigatórios, _exceto_: Responsável Principal, Descrição da Etapa e Anexos.&#x20;
{% endhint %}

<figure><img src="../../../.gitbook/assets/xx5 (2).png" alt=""><figcaption></figcaption></figure>

{% hint style="danger" %}
Só é possível inserir datas que estejam entre a data de início e a data final do projeto.&#x20;
{% endhint %}

Um dos campos obrigatórios é o de _<mark style="color:$info;">Status</mark>_, que terá que ser atualizado conforme o andamento do projeto.&#x20;

Os status definidos para as etapas do cronograma são:&#x20;

* Não Iniciado
* Em Andamento
* Travado
* Concluído

{% hint style="warning" %}
É obrigatório informar o motivo pelo qual uma etapa é marcada inicialmente com o status “Travado”:
{% endhint %}

<figure><img src="../../../.gitbook/assets/xx10 (2).png" alt=""><figcaption></figcaption></figure>

Ainda, há os campos de _<mark style="color:$info;">Área Responsável</mark>_ e _<mark style="color:$info;">Responsável Principal</mark>_.&#x20;

Importante apontar que a _<mark style="color:$info;">Área Responsável</mark>_ não precisa ser necessariamente a área que cadastrou o projeto nem uma área corresponsável do projeto, pode ser **qualquer área**.&#x20;

Isso porque pode ser que uma área seja responsável apenas por uma única etapa do projeto.&#x20;

{% hint style="info" %}
Por exemplo, em um projeto cuja área responsável seja CPI e CPDDH seja uma área corresponsável, pode ser que a Assessoria Jurídica (AJ) seja responsável por apenas uma etapa. Nesse caso, CPI e CPDDH poderão visualizar e editar todo o projeto e todas as etapas e a AJ poderá visualizar e editar apenas a _etapa_ pela qual ela é responsável.&#x20;
{% endhint %}

Por fim, o SIAD permite que sejam anexados documentos relacionados à etapa. Se trata de uma funcionalidade <mark style="background-color:yellow;">opcional</mark>, voltada para que as áreas possam acessar recursos que facilitem a gestão do cronograma de seus projetos.

Basta, no campo indicado, arrastar o documento que deseja incluir ou selecionar em seus arquivos.

Além dos anexos das etapas, há como incluir anexos ~~do~~ projeto como um todo. Para mais informações, acesse a página de [Anexos](../anexos.md).

Após o preenchimento, é possível clicar em _<mark style="color:purple;">Criar</mark>_ para salvar a etapa ou em _<mark style="color:$info;">Salvar e criar outro</mark>_ para salvar a etapa e adicionar uma nova em sequência.&#x20;
